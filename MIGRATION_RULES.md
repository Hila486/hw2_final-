# HW2 Migration Rules

- `hw2_new_skeleton` is the final project and source of truth.
- `hw2_old_skeleton` is reference only.
- Do not copy old headers into the new skeleton.
- Do not change public headers, method signatures, data types, executable names, or run formats unless explicitly approved.
- Cursor must analyze before editing.
- Migrate one component at a time.
- After each component: build, test if possible, commit.
- Before final submission, remove `hw2_old_skeleton` from the project folder.
